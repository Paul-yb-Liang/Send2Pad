# Send2Pad — 技术文档

## 编译说明

### 环境要求

- MinGW-w64（gcc 支持 `-mwindows`）
- GNU Make（可选，也可直接用 gcc 命令）
- UPX（可选，用于压缩体积）

### 编译命令

```bash
# 完整编译（含图标资源）
windres app_icon.rc -O coff -o app_icon.o
gcc -O1 -s -Wall -DUNICODE -D_UNICODE -DWIN32_LEAN_AND_MEAN \
  -DSTATIC_IN_RELEASE=static \
  -DMAJOR_VERSION=4 -DMINOR_VERSION=1 -DMICRO_VERSION=1 -DVERSION='"4.1.1"' \
  -Iinclude -Ivendor/libqrencode-4.1.1 \
  app_icon.o src/*.c \
  vendor/libqrencode-4.1.1/qrencode.c \
  vendor/libqrencode-4.1.1/qrinput.c \
  vendor/libqrencode-4.1.1/qrspec.c \
  vendor/libqrencode-4.1.1/split.c \
  vendor/libqrencode-4.1.1/mask.c \
  vendor/libqrencode-4.1.1/mmask.c \
  vendor/libqrencode-4.1.1/rsecc.c \
  vendor/libqrencode-4.1.1/bitstream.c \
  vendor/libqrencode-4.1.1/mqrspec.c \
  -o Send2Pad.exe -mwindows \
  -lws2_32 -lgdi32 -luser32 -lshell32 -lshlwapi -liphlpapi
```

或使用 Makefile：

```bash
make all  # 编译
make clean  # 清理
```

### libqrencode 编译说明

libqrencode 以源码形式内嵌于 `vendor/libqrencode-4.1.1/`。编译时需定义以下宏：

| 宏 | 值 | 说明 |
|---|-----|------|
| `STATIC_IN_RELEASE` | `static` | 使内部函数在最终 exe 中不出现在符号表 |
| `MAJOR_VERSION` | `4` | 版本号 |
| `MINOR_VERSION` | `1` | 版本号 |
| `MICRO_VERSION` | `1` | 版本号 |
| `VERSION` | `"4.1.1"` | 版本字符串 |

需要编译的源文件清单见 Makefile 中的 `QRCS` 变量，包含了 `mmask.c`（用于 Micro QR 码，否则链接报错）。

## 模块详解

### 1. 命令行解析（`parse_cmdline.c`）

- 接收 `WinMain` 的 `lpCmdLine`（ANSI）→ 改用 `GetCommandLineW()`（宽字符）
- 处理引号包裹的路径（`"C:\path with spaces\file.txt"`）
- 用 `PathFileExistsW()` 验证路径存在性
- 排除目录类型

### 2. 文件验证与 MIME（`file_utils.c`）

- `GetFileAttributesW()` 检查文件存在性和非目录
- MIME 表覆盖 16 种常见文件类型（txt/pdf/jpg/png/gif/bmp/zip/mp4/mov/doc/docx/xls/xlsx/html/htm/xml）
- 扩展名匹配不区分大小写（`towlower` 转换后查表）
- 无法识别时返回 `application/octet-stream`

### 3. 临时目录管理（`temp_utils.c`）

- 用 `GetTempPathW()` 获取系统临时目录路径
- 用系统时间 + 随机后缀生成唯一目录名（`Send2Pad_20260527_123045_abcd`）
- `CopyFileW()` 复制源文件
- `CleanupTempWorkspace()` 递归删除目录（`FindFirstFileW`/`FindNextFileW` 遍历）

### 4. 网络初始化（`network.c`）

**GetLocalIP**：UDP 探测法。`connect` 到 8.8.8.8:80（不发数据包），然后 `getsockname` 获取本机 IP。

**GetAllLocalIPs**：`GetAdaptersAddresses()` 枚举所有活跃非回环网卡（排除 `IF_TYPE_SOFTWARE_LOOPBACK` 和 `OperStatus != IfOperStatusUp`）。

**BindRandomPort**：`bind(s, port=0)` → 系统分配随机端口 → `getsockname` 读取实际端口。

**UrlEncode**：宽字符 → `WideCharToMultiByte(CP_UTF8)` → 对非 unreserved 字符做 percent-encoding。

### 5. HTTP 服务（`http_server.c`）

手搓 HTTP/1.1 GET 服务器：

1. `listen()` → `accept()`（单连接循环，404 时返回继续 `accept`）
2. `recv()` 读请求行 → `sscanf` 解析 `GET /path HTTP/1.x`
3. 文件名匹配（大小写不敏感，先做 percent-decode 再比较）
4. 匹配成功 → 组装响应头（`Content-Disposition: attachment; filename*=UTF-8''...`）
5. `ReadFile()` + `send()` 64KB buffer 流式发送
6. 发完 → `PostMessage(hwnd, WM_HTTP_DONE)` → 关闭 listen socket → 线程退出

### 6. 二维码生成（`qrcode.c`）

**GenerateQRBitmapAutoFit**（自动适配）：

1. `QRcode_encodeString()` → 获取 QR 模块数（`qrcode->width`）
2. 计算最佳 pixelSize：`pixelSize = maxSize / (width + 2*border)`
3. 用计算的 pixelSize 调用 `GenerateQRBitmap()`

**GenerateQRBitmap**：

1. 创建 DIBSection（32-bit BGRA，`height = -bmpSize` 表示从上到下）
2. 整体先填白色（形成 quiet zone）
3. 在中间区域绘制 QR 模块（`border * pixelSize` 偏移）
4. 每个模块展开为 `pixelSize × pixelSize` 的方块

### 7. GUI 窗口（`window.c`）

Win32 API 手写窗口：

- 窗口类使用 `WNDCLASSEXW`（支持 `hIcon` 和 `hIconSm` 大小图标）
- `AdjustWindowRect` 精确计算窗口尺寸
- 控件：STATIC（状态文字）、STATIC（SS_BITMAP 显示 QR 码）、EDIT（只读多行显示 URL）
- 自定义消息：
  - `WM_HTTP_READY`：传递 HBITMAP 和 URL 给窗口
  - `WM_HTTP_DONE`：HTTP 传输完成，弹框询问
  - `WM_DESTROY`：关闭 listen socket 唤醒 HTTP 线程
- `WM_CTLCOLORSTATIC` + `WM_CTLCOLOREDIT` 统一白色背景

### 8. 主流程串联（`main.c`）

14 步顺序执行：

1. 解析命令行（跳过 exe 自身路径）
2. 验证文件路径
3. 获取 MIME 类型
4. 创建临时工作目录
5. 复制文件到临时目录
6. 初始化 Winsock
7. 枚举网卡（多网卡时弹框提示）
8. 绑定随机端口
9. URL 编码 + 构建完整 URL
10. 生成二维码（自动适配控件尺寸）
11. 注册窗口类
12. 初始化 HTTP 上下文
13. 创建主窗口（显示 QR + URL）
14. 启动 HTTP 线程 → 进入消息循环

## 关键设计决策

### 为何选择纯 C + Win32 API 而非 C++

- 体积极致压缩（最终 59KB）
- 功能足够简单，不需要 C++ 的高级特性
- 对内存和行为有完全控制力

### 为何手搓 HTTP 而非用库

- 只需要一个 GET 端点，约 200 行代码
- 零外部 HTTP 依赖，控制体积

### 为何用 GetAdaptersAddresses 而非旧 API

- `GetAdaptersAddresses()` 是 IPv4+IPv6 兼容的现代 API
- 比 `GetAdaptersInfo()` 更准确，支持更丰富的状态信息

### 二维码自动适配策略

- 根据 QR 版本动态计算 pixelSize（不是固定值）
- 短文件名（V3-V4）→ pixelSize=12，QR 大而清晰
- 长文件名（V6-V8）→ pixelSize 自动缩小，完整显示
- 任何长度都能完整显示，只影响清晰度
