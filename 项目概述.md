# Send2Pad — 项目概述

## 项目背景

原有一个基于 Python（tkinter + qrcode + Pillow + http.server）的「Send to iPad」工具，功能完整但存在三个痛点：

| 问题 | Python 版 | C 版目标 |
|------|----------|---------|
| 体积 | 10~20MB（含 Python 运行时） | < 150KB（单 exe） |
| 启动速度 | 1~2 秒（Python 解释器加载） | < 100ms |
| 运行依赖 | 需安装 Python + 第三方库 | 零依赖（仅 Windows DLL） |

## 项目目标

用 C 语言重写，产出单一 `.exe` 文件。核心功能与原版一致：

1. **拖放文件**到 exe → 解析文件路径
2. 启动**HTTP 服务**（随机端口）→ 提供文件下载
3. 生成**二维码** → 显示在窗口中
4. iPad 扫码 → 浏览器下载文件
5. 传输完成 → 弹框确认 → 清理临时文件并退出

## 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 语言 | C11 | 零运行时开销 |
| 编译器 | MinGW-w64 gcc 15.2 | Windows 原生编译 |
| HTTP | 手搓（WinSock2） | 仅支持 GET，约 200 行 |
| 二维码 | libqrencode 4.1.1（源码内嵌） | C 库，~30KB |
| GUI | Win32 API | CreateWindowEx + GDI |
| 多线程 | CreateThread | HTTP 服务线程 + 主消息泵 |
| 文件传输 | ReadFile + send() 循环 | 64KB buffer 流式发送 |

## 架构图

```
[WinMain]
  │
  ├─ 解析拖放路径（ParseCommandLine）
  ├─ 验证文件 + MIME 类型（ValidateFilePath / GetMimeType）
  ├─ 创建临时目录 + 复制文件（CreateTempWorkspace / CopyFileToTemp）
  ├─ 初始化 Winsock + 探测 IP + 绑定端口（InitWinsock / GetLocalIP / BindRandomPort）
  ├─ URL 编码 + 生成二维码（UrlEncode / GenerateQRBitmapAutoFit）
  ├─ 创建 GUI 窗口（RegisterAppClass / CreateMainWindow）
  └─ 启动 HTTP 线程（StartHttpThread）
       │
       ▼
  [消息循环]  ←── PostMessage(WM_HTTP_DONE) ←── [HTTP 线程]
       │                                          │
       ▼                                          ├─ listen()
  WM_DESTROY                                      ├─ accept()
  → closesocket(listenSock)                        ├─ recv() 解析 GET
  → PostQuitMessage                                ├─ send() 文件流
       │                                          └─ PostMessage(WM_HTTP_DONE)
       ▼
  [清理] WaitForThread → CleanupTempWorkspace → WSACleanup → exit
```

## 最终指标

| 指标 | 数值 |
|------|------|
| 最终体积 | ~59KB（strip 后，未压缩） |
| 源代码行数 | ~1500 行（9 个 .c + 8 个 .h） |
| 启动时间 | < 50ms |
| 运行时内存 | < 2MB |
| 外部依赖 | 无（静态链接 libqrencode + Win32 API） |
