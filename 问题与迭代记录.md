## 1. MinGW 编译兼容性问题

**现象**：初始编译失败，`ws2tcp.h` 找不到，`#pragma comment` 警告。

**原因**：
- `ws2tcp.h` 是 MSVC 命名，MinGW 中为 `ws2tcpip.h`
- `#pragma comment(lib, ...)` 是 MSVC 指令，MinGW 不认识

**解决**：
- `network.h`: `ws2tcp.h` → `ws2tcpip.h`，移除 `#pragma comment`
- `parse_cmdline.c`: 移除 `#pragma comment(lib, "shlwapi.lib")`
- 改为在 Makefile 的 `LDFLAGS` 中链接库

**涉及文件**：`include/network.h`, `src/parse_cmdline.c`

## 2. libqrencode 编译错误

**现象**：`STATIC_IN_RELEASE` 未定义、`MAJOR_VERSION` 未定义、`MMask_mask` 链接失败。

**原因**：libqrencode 默认需要 CMake/autotools 生成的 config.h，我们直接编译源码需要手动定义这些宏。

**解决**：
- 添加编译参数：`-DSTATIC_IN_RELEASE=static -DMAJOR_VERSION=4 -DMINOR_VERSION=1 -DMICRO_VERSION=1 -DVERSION='"4.1.1"'`
- 补编译 `mmask.c`（之前遗漏，链接 `MMask_mask` 需要）

**涉及文件**：`Makefile`（更新了 CFLAGS 和 QRCS 变量）

## 3. 二维码无法被 iPad 扫描

**现象**：窗口弹出二维码，但 iPad 相机扫描不出链接。

**原因**：**两个问题叠加**

1. **缺少空白边（quiet zone）**：QR 规范要求四周至少 4 模块宽的白色边框。libqrencode 的 `QRcode_encodeString()` 只返回 QR 码矩阵，不自动添加边框。原代码直接把矩阵画满整个位图，finder pattern 无法被检测。

2. **控件尺寸偏小**：250x250 的静态控件，pixelSize=8 时版本 4 的 QR 码（33×8=264px）超出控件范围，底部定位图案被裁剪。

**解决**：
- `GenerateQRBitmap()` 新增 `border` 参数，先整图填充白色，再将 QR 数据居中绘制
- QR 静态控件从 250×250 扩大到 400×400
- 窗口改用 `AdjustWindowRect` 精确计算尺寸

**涉及文件**：`src/qrcode.c`, `include/qrcode.h`, `src/window.c`, `src/main.c`

## 4. 关闭窗口时的优雅退出

**问题**：用户提前关闭窗口时，主线程硬等 5 秒后暴力清理，可能导致：
- 传输中的文件被截断（iPad 收到残缺文件）
- `WSACleanup()` 与 HTTP 线程的 `send()` 并发冲突

**解决**：
- `WM_DESTROY` 中先关闭 `listenSock` → HTTP 线程的 `accept()` 立即返回错误 → 线程退出
- 主线程改为 `WaitForSingleObject(hThread, INFINITE)` 等待线程自然结束
- 传输中 → 线程走完 `send()` 循环后自动退出 → 传输完整
- 未开始 → socket 被关 → `accept()` 报错 → 线程立即退出

**涉及文件**：`src/main.c`, `src/window.c`

## 5. 第一次二维码显示不全修复后仍有部分文件超出

**现象**：扩大 QR 控件到 400×400 后，大部分文件正常，但文件名特别长的仍会被截断。

**原因**：固定 pixelSize=8，URL 过长 → QR 版本升高 → V7(424px) 超出 400px 控件。

**解决**：新增 `GenerateQRBitmapAutoFit()` 函数——先编码获取 QR 模块数，再根据控件尺寸动态计算最佳 pixelSize，确保任何长度的 URL 都能完整显示。短文件名 pixelSize=12（更清晰），长文件名自动缩小。

**涉及文件**：`src/qrcode.c`, `include/qrcode.h`, `src/main.c`

---

## 6. URL 标签不换行

**现象**：窗口底部的 URL 文字显示不全，长 URL 被截断。

**原因**：STATIC 控件单行显示，超过宽度的文字直接裁剪。

**解决**：将 URL 标签从 `STATIC`（`SS_CENTER`）改为 `EDIT`（`ES_MULTILINE | ES_READONLY`），支持自动换行。同时增加标签高度从 40px 到 60px，添加 `WM_CTLCOLOREDIT` 保持白色背景。

**涉及文件**：`src/window.c`

---

## 7. 图标小图标显示错误

**现象**：大图标和中图标正确，小图标（任务栏/标题栏）显示异常。

**原因**：
- 第一次：ICO 文件缺少 16×16 尺寸 → 添加后修复
- 第二次：`LoadImageW(..., LR_DEFAULTCOLOR)` 导致 PNG 图标 alpha 通道丢失 → 改用 `LoadIconW()`
- 最终问题：Windows Explorer 图标缓存未刷新，显示旧版蓝色图标

**解决**：
- ICO 包含 6 个尺寸（16/32/48/64/128/256）
- `hIconSm` 改用 `LoadIconW()` 由系统自动选尺寸
- 如仍显示旧图标，运行 `ie4uinit.exe -ClearIconCache` 或重启 Explorer

**涉及文件**：`src/window.c`, `app_icon.ico`, `app_icon.rc`

## 9. 嵌入图标时的依赖工具

ICO 文件生成需要 Pillow（Python 图像库）。在开发环境中的安装方式：

```bash
python -m venv env_icon
env_icon/Scripts/pip install Pillow
env_icon/Scripts/python -c "
from PIL import Image
img = Image.open('source.png').convert('RGBA')
sizes = [(16,16), (32,32), (48,48), (64,64), (128,128), (256,256)]
icons = [img.resize(s, Image.Resampling.LANCZOS) for s in sizes]
icons[0].save('app_icon.ico', format='ICO', sizes=sizes, append_images=icons[1:])
"
```
